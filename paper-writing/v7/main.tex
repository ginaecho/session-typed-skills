\begin{abstract}
Skill files, agent markdowns, and specification documents have become the de facto programming language of multi-agent systems: they declare who talks to whom, in what order, carrying what payloads, under which authorization obligations. Unlike every other programming language, this one ships without a compiler. Whether a set of skills can deadlock, leak a confidential payload to an unauthorized role, or ever reach its goal is today discoverable only at runtime---by running the system, or by hiring further LLM agents to judge its traces---and neither can establish the \emph{absence} of failure: a thousand clean runs warrant nothing about run 1{,}001. We present \stjp{}, a static compiler for agent interactions grounded in multiparty session types (MPST). A user intent is drafted by an LLM into a Scribble global protocol; the protocol is validated \emph{before any agent runs}, establishing deadlock-freedom, communication safety, session fidelity, bounded recursion, and goal reachability by construction; the validated global type is then projected into per-role local contracts (lean skills) and mechanically compiled into deterministic finite-state monitors, value-dependent choice guards, and an EFSM scheduler---code, not agents. Across six pre-registered experiments on finance and banking pipelines (gpt-4o and gpt-5.4, up to seven ablation arms), the full stack reaches 100\% strict goal completion with zero irreversible-action disasters, while intent-only agents commit up to 22 unauthorized irreversible acts in 30 attempts---\emph{more} with the stronger model. Enforcement plus protocol-driven scheduling delivers the same guarantees at $9\times$ lower cost-to-goal and $-73\%$ LLM calls than protocol-as-prompt-text on the same runtime; a seeded circular-wait protocol is rejected statically at zero token cost, a failure class no runtime method can rule out. A pre-registered validation suite then scales the evidence: mutation testing shows the checker detects 95.1\% of seeded well-formedness defects with \emph{zero} false positives over a 100-protocol corpus; against 1{,}200 prompt-injected exfiltration attempts, keyword rules block 41.7\% while the structural gate plus value refinements block 100\%; and a six-arm ladder at $n{=}100$ on two further tasks (1{,}200 trials) shows every \emph{observing} arm incurring real violations---unenforced protocol text reaches the goal in 100\% of runs yet commits an unauthorized irreversible act in 95 of them under concurrent scheduling---while every \emph{enforcing} arm shows zero by construction, with the full stack simultaneously the safest and the cheapest arm. A measured three-tier capability sweep then locates the boundary of prompting: unenforced safety is capability-dependent (95 premature filings per 100 at the weak tier, 0 at the mid and strong tiers, on identical protocol text) whereas enforcement is capability-\emph{invariant}---0 disasters at every tier---which is the difference between a behavior one hopes for and a property one has. Three typed extensions, each licensed by a session-type result, are implemented and measured: stateful session invariants catch cumulative-budget violations that per-message guards are structurally blind to (0/50 $\to$ 50/50, zero false positives), a precise-subtyping check turns contract compression into a decidable build step, and statically verified crash handling converts ``no session left in limbo'' from an incident report into a compile-time property (crash grid: 100\% limbo $\to$ 100\% typed termination, with all five seeded unsafe recovery handlers rejected). We report our falsified predictions prominently, formalize \emph{conformance} and \emph{goal achievement} over a typed event log that needs neither golden trajectories nor LLM judges, and release the suite as a benchmark for statically verified agent specifications.
\end{abstract}

\section{Introduction}\label{sec:intro}
Practitioners now steer LLM agents with markdown artifacts: Anthropic's Agent Skills open standard (\texttt{SKILL.md}), \texttt{AGENTS.md} conventions, OpenAI's Model Spec, MCP tool manifests, and enterprise ``agent spec'' documents \citep{anthropic-skills}. Functionally, these artifacts are programs. They fix an interaction discipline---which role sends which message to which role, in what order, carrying what payload, with which authorization preceding which irreversible act. When several agents must cooperate (a planner, an analyst, an auditor, a filer), the artifact is not a prompt; it is a \emph{protocol}.

Yet the toolchain that grew around ordinary programming languages is absent. There is no parser that rejects an incoherent skill, no type checker that proves two skills cannot mutually block, no compiler that guarantees the declared goal is reachable. Format linting exists---Skilldex reports that 34\% of community skill packages violate even the published format specification \citep{skilldex}---but format validity says nothing about interaction soundness. A \texttt{SKILL.md} can parse perfectly while describing a choreography in which the planner waits for a result the worker will never send, because the worker is waiting for an answer the planner cannot give. Nothing crashes; the system sits there, emitting tokens.

The field's current answers are all \emph{dynamic}. Runtime guardrails such as AgentSpec \citep{agentspec} and Agent-C \citep{agentc} intercept unsafe tool calls with rule DSLs or SMT-checked temporal constraints; benchmarks such as ST-WebAgentBench \citep{stweb} and BeSafe-Bench \citep{besafe} measure, after the fact, how often agents violated policy; LLM-as-judge pipelines score traces with a second model whose own reliability is contested (reported true-negative rates below 25\% and expert agreement of 64--68\% \citep{judgebias}). All of these \emph{observe executions}. None can certify a \emph{specification}. Testing, as Dijkstra observed, shows the presence of bugs, never their absence.

This paper poses the question in its static form: \textbf{can we type-check the conversation itself, before any agent runs?} Our answer is \stjp{} (Session-Typed skills/Judge Protocol), a compiler pipeline built on two decades of multiparty session type theory \citep{hyc08,scalas19} and its industrial protocol language, Scribble \citep{scribble}. The user states an intent; an LLM drafts a Scribble global protocol; the Scribble checker validates well-formedness statically---rejecting deadlock precursors, incoherent choice, unguarded recursion, unreachable goals---and only a validated protocol is projected, role by role, into local contracts that become each agent's lean skill. From the same projection we mechanically generate a deterministic finite-state monitor per role, value-dependent \emph{choice guards} in the asserted-MPST style \citep{bocchi10}, and an EFSM scheduler. Enforcement and evaluation are plain code: bit-reproducible, token-free, judge-free.

\paragraph{Contributions.}
\begin{enumerate}\itemsep2pt
\item \textbf{A reframing.} Skills, specs, and agent markdowns constitute an untyped protocol language; we give, to our knowledge, the first end-to-end pipeline that statically type-checks them via MPST before execution, and we make the correspondence precise (\S\ref{sec:mpst}--\S\ref{sec:system}).
\item \textbf{Guarantee transfer.} We restate the MPST metatheory---communication safety, session fidelity, deadlock-freedom, monitorability, refinement, gradual typing of the LLM endpoint---as agent-system properties with direct safety and cost consequences (\S\ref{sec:mpst}, Table~\ref{tab:guarantees}).
\item \textbf{Judge-free evaluation.} Because every run is, by construction, a path through a validated global type, \emph{conformance} and \emph{goal achievement} receive mechanical definitions over a typed event log---no golden trajectory, no LLM judge---together with a consequence-graded violation taxonomy (S0--S4) that we validate empirically: $P(\text{goal failure}\mid\text{S2+ violation})=100\%$ in every arm of every run (\S\ref{sec:method}).
\item \textbf{Evidence, honestly graded.} Six experiments across two tasks, two models, and up to seven ablation arms (\S\ref{sec:results}, Fig.~\ref{fig:results}): 0\%$\to$80--100\% strict goal completion along the protocol-information ladder; zero disasters in every typed arm versus 22 unauthorized irreversible acts by intent-only agents under the \emph{stronger} model; a statically rejected deadlock; $9\times$ cost-to-goal reduction from EFSM scheduling. We also report the run in which unenforced protocol text \emph{ties} the enforcement gate, and what that bounds.
\item \textbf{A benchmark agenda.} A concrete proposal---protocol-mutation testing, adversarial injection, model-strength sweeps, $\mathrm{pass}^k$ reliability, translation-fidelity measurement---for evaluating static agent verification as a field (\S\ref{sec:bench}).
\end{enumerate}

\section{Related Work}\label{sec:related}
\textbf{Agent governance layers.} Recent syntheses organize agent governance as a three-layer stack \citep{governspec,lga}: Layer~0 checks that a specification parses; Layer~1 asks whether the specification is semantically sound (``if the agent believes the spec, will it be correct?''); Layer~2 measures whether runtime behavior matches claims. Layer~0 is commoditized \citep{skilldex}. Layer~2 is an active benchmark frontier: the best of 13 agents evaluated by BeSafe-Bench completes fewer than 40\% of tasks while fully respecting safety constraints \citep{besafe}; ST-WebAgentBench's Completion-under-Policy shows policy-respecting completion at under two-thirds of nominal completion \citep{stweb}. Layer~1 is, in those surveys' words, ``not available.'' \stjp{} occupies it.

\textbf{Runtime enforcement.} AgentSpec \citep{agentspec} enforces trigger/predicate/action rules around tool calls and prevents $>$90\% of unsafe executions in its evaluation; Agent-C \citep{agentc} compiles temporal constraints to SMT checks interleaved with generation; LGA \citep{lga} composes sandboxing, intent verification, zero-trust authorization, and audit logging. All act per execution. Rule DSLs cannot express, let alone verify, \emph{global} interaction properties---deadlock-freedom, goal reachability, choice coherence---because these are properties of the protocol structure, not of any single call. \stjp{} rejects the faulty specification before the first LLM call, the only point at which a guarantee can be universal rather than per-run; its runtime gate then enforces exactly the properties the static phase proved enforceable \citep{bocchi13}. We are explicit about where the novelty lies: the session-type theory we build on is three decades deep and is assembled here, not invented---our contributions are the compiler framing (skills \emph{are} an untyped protocol language; \stjp{} is its missing compiler), the gradual-typing integration of an untyped LLM participant, the observation that the \emph{scheduler} is itself a projection artifact, and the judge-free empirical program the rest of this paper executes.

\textbf{Session types and choreographies.} MPST \citep{hyc08,scalas19} and its toolchain Scribble \citep{scribble} give protocols a type theory (\S\ref{sec:mpst}); choreographic programming \citep{montesi23,carbone13} generates endpoint code from a global script by projection. An \texttt{AGENTS.md} is an informal choreography; \stjp{} makes it formal and lets projection do the work. Prior NL$\to$formal-spec pipelines (NL2Spec \citep{nl2spec}, PROSPER \citep{prosper}) target LTL and network RFCs; we target a protocol algebra whose checker exists and whose theorems transfer.

\textbf{Trace evaluation.} Tool-and-step metrics (LangSmith, DeepEval, Phoenix, MLflow), process-utility metrics (TRACE \citep{trace}), policy-conditioned metrics (CuP \citep{stweb}), and reliability metrics ($\mathrm{pass}^k$) all presuppose a hand-authored reference trajectory or an LLM judge, because an unconstrained agent leaves no structure to score a run against. Under \stjp{} the validated global type \emph{is} the reference (\S\ref{sec:method}).

\section{Multiparty Session Types: Syntax, Semantics, Guarantees}\label{sec:mpst}
We summarize the fragment of MPST that \stjp{} compiles to, in the modern formulation \citep{hyc08,scalas19}; this section fixes notation and states (without proof) the theorems whose transfer to agents Table~\ref{tab:guarantees} prices.

\subsection{Global and local types}
Fix a set of \emph{roles} $p,q,r\in\mathcal{R}$ (agents, tools, the orchestrator), \emph{labels} $\lbl\in\mathcal{L}$ (message kinds), and \emph{payload sorts} $U$ (base sorts and refinements, below). \emph{Global types} describe the whole conversation from a bird's-eye view; \emph{local types} describe one role's view of it:
\begin{align*}
\gtG \;::=\;& p \sends q : \{\lbl_i(U_i).\,\gtG_i\}_{i\in I} \;\mid\; \mu t.\,\gtG \;\mid\; t \;\mid\; \ed
&& \text{(global types)}\\
\lt \;::=\;& q\,{\oplus}\,\{\lbl_i(U_i).\,\lt_i\}_{i\in I} \;\mid\; p\,\&\,\{\lbl_i(U_i).\,\lt_i\}_{i\in I} \;\mid\; \mu t.\,\lt \;\mid\; t \;\mid\; \ed
&& \text{(local types)}
\end{align*}
$p \sends q:\{\lbl_i(U_i).\gtG_i\}_{i}$ reads: $p$ chooses one label $\lbl_i$, sends it with a payload of sort $U_i$ to $q$, and the conversation continues as $\gtG_i$. Dually, $q\oplus\{\ldots\}$ is an \emph{internal choice} (this role selects and sends to $q$) and $p\,\&\,\{\ldots\}$ an \emph{external choice} (this role branches on what it receives from $p$). $\mu t.\gtG$ is guarded recursion (loops); $\ed$ terminates the session. In the agent reading: labels are message kinds (\texttt{ExpenseData}, \texttt{Approval}), sorts are payload schemas, branching is a decision (audit vs.\ standard path), recursion is a bounded retry/deliberation loop.

\subsection{Projection}
The \emph{projection} $\proj{\gtG}{r}$ extracts role $r$'s local type from $\gtG$:
\[
\proj{\big(p \sends q : \{\lbl_i(U_i).\gtG_i\}_{i\in I}\big)}{r} \;=\;
\begin{cases}
q\,{\oplus}\,\{\lbl_i(U_i).\,\proj{\gtG_i}{r}\}_{i\in I} & \text{if } r=p\\[2pt]
p\,\&\,\{\lbl_i(U_i).\,\proj{\gtG_i}{r}\}_{i\in I} & \text{if } r=q\\[2pt]
\sqcap_{i\in I}\, \proj{\gtG_i}{r} & \text{otherwise,}
\end{cases}
\]
with $\proj{(\mu t.\gtG)}{r}=\mu t.(\proj{\gtG}{r})$ when $r$ occurs in $\gtG$ (else $\ed$), $\proj{t}{r}=t$, $\proj{\ed}{r}=\ed$. The \emph{merge} $\merge$ requires that a role not involved in a choice either behaves identically on all branches or is informed of the outcome before its behavior may differ; if the merge is undefined, $\gtG$ is \emph{rejected as unprojectable}---already a useful lint: it flags protocols in which an agent's duties silently depend on a decision it was never told about. A global type is \emph{well-formed} if it is projectable onto every role and all recursion is guarded. Each local type induces a finite \emph{endpoint state machine} (EFSM) $M_r=(S,s_0,\Sigma,\delta,F)$ whose alphabet is $r$'s sends and receives; conversely, under well-formedness, the (associative) composition of the projected EFSMs recovers the behaviors of $\gtG$ \citep{denielou12}.

\subsection{Operational semantics and typing}
A \emph{configuration} $C=(P_1,\ldots,P_n;\,\vec h)$ pairs role processes with FIFO queues $h_{pq}$ (asynchrony). The labelled transition system has actions $pq!\lbl(v)$ (role $p$ enqueues $\lbl(v)$ for $q$) and $pq?\lbl(v)$ (role $q$ dequeues), and $\mathrm{traces}(C)$ collects its finite action sequences. The typing judgment $\Gamma \vdash C \,\triangleright\, \Delta$ assigns each role's process its local type ($\Delta = \{p_i : \lt_i\}$); in the ``Less is More'' formulation \citep{scalas19}, desired behavioral properties are checked directly as safety/liveness predicates on the typing environment $\Delta$, repairing soundness gaps of the classical presentation and enlarging the set of typable protocols. Well-typedness is preserved under reduction (subject reduction), which drives:

\begin{itemize}\itemsep1pt
\item[\textbf{T1}] \textbf{(Communication safety.)} In a well-typed configuration, no role ever receives a message whose label or payload sort it cannot handle at its current state: every send is matched by a compatible receive \citep{hyc08}.
\item[\textbf{T2}] \textbf{(Session fidelity / protocol conformance.)} Every trace of a well-typed configuration is a trace of $\gtG$: execution \emph{refines} the global type \citep{hyc08,scalas19}.
\item[\textbf{T3}] \textbf{(Deadlock-freedom / progress.)} A well-typed single-session configuration never reaches a stuck non-final state: some role can always move until $\ed$ \citep{hyc08,scalas19}.
\item[\textbf{T4}] \textbf{(Monitorability.)} If every endpoint is wrapped by a monitor that enforces its projected local type $\proj{\gtG}{r}$---accepting, suppressing, or rejecting boundary messages---then the observable behavior of the whole (untrusted) network satisfies $\gtG$; local checks compose to a global guarantee with no central observer \citep{bocchi13}.
\end{itemize}
T4 is the license for \stjp{}'s architecture: LLM endpoints are untrusted, monitors are generated from projections, and \emph{global} compliance falls out of \emph{local}, $O(1)$-per-event checking.

\subsection{Extensions \stjp{} uses}
\textbf{Refinements and choice guards.} Asserted/refined MPST \citep{bocchi10,refine24} decorates payloads with predicates, $\lbl(x{:}U)\{A\}$, where $A$ may mention previously bound values; \citet{chen12} extend this line to \emph{stateful} asynchronous properties---assertions over virtual state that evolves across the whole conversation, checkable at endpoints under asynchrony---implemented and measured as \stjp{}'s \emph{session ledger} (\S\ref{sec:ext}); and decorates \emph{choices} with guards selecting which branch the data permits:
$p \sends q : \{\,\lbl_i(x_i{:}U_i)\{A_i\}.\,\gtG_i\,\}_{i}$ with $A_i$ over the session history. Statically dischargeable guards go to an SMT check; guards over LLM-produced values compile into the monitor. This is precisely the construct that closes the ``protocol-legal but value-wrong'' hole we observed empirically (\S\ref{sec:results}): choosing the \texttt{Standard} branch on \$50k+ revenue is legal for classical MPST and a violation for guarded MPST.

\textbf{Behavioral flexibility via precise subtyping.} An agent need not follow its projected local type \emph{exactly}: session subtyping characterizes every safe deviation, and its \emph{precise} (sound \& complete) characterization \citep{chen17,chen24} draws the line tightly---every subtype-conformant behavior is safe, and any behavior outside the relation fails in some context. This is the formal license behind our lean contracts (\S\ref{sec:results7}): a compressed contract refined by the verbose one loses nothing, and preciseness says exactly how much slack a monitor may lawfully grant before it must reject; both directions---a compile-time subtype check and a tolerance-bounded gate---are implemented and measured in \S\ref{sec:ext}. \textbf{Gradual typing of the LLM endpoint.} Gradual session types \citep{igarashi17} mix statically typed participants with a dynamically typed ($\mathtt{dyn}$) one, inserting casts at the boundary that preserve safety and progress. The LLM is \stjp{}'s $\mathtt{dyn}$ participant; the generated monitor \emph{is} the cast. The theory predicts what we measure: the untyped participant may \emph{attempt} anything; nothing ill-typed crosses the boundary.

\textbf{Beyond the fragment.} Parameterised MPST \citep{param10} types fan-out over $n$ workers; dynamic multirole session types \citep{denielou11} admit roles joining/leaving---both on our roadmap, neither exercised by the experiments below, which fix the role set.

\subsection{Pipeline}\label{sec:pipeline}
\textbf{S1 --- Intent $\to$ draft protocol.} The user states a task (``six agents process a revenue report; if revenue exceeds \$50{,}000 an audit is mandatory before filing''). An LLM drafts a Scribble global protocol---roles, labels, payload sorts, branches, guarded recursion---plus a sidecar of refinement and choice guards (\texttt{.refn}; stock \texttt{scribble-java} is untouched) and \emph{goal markers}: designated protocol points whose ordered satisfaction encodes task completion, one marked \textsc{final}. The draft is endorsed by the user; intent negotiation is human-in-the-loop by design.

\textbf{S2 --- Static validation.} The Scribble checker validates well-formedness: projectability (choice coherence via the merge $\merge$), matched sends/receives, guarded recursion, and reachability of every goal marker (the achievability check emits a witness path, later reused as the efficiency baseline). Validation is not decorative: live-drafting the banking case, four consecutive LLM drafts were rejected before the fifth passed, and the rejected finance draft contained a wait-for cycle $[\textit{Writer}\to\textit{RevenueAnalyst}\to\textit{TaxVerifier}]$ that \emph{would} deadlock at runtime---caught before any agent ran. Agents run on that unsafe draft completed only 20\% of trials.

\textbf{S3 --- Projection $\to$ local contracts.} $\proj{\gtG}{r}$ is rendered as a lean per-agent skill: a \textsc{send}/\textsc{recv} table over the role's EFSM states with its refinement and choice guards attached at the states where the decision is made. The compressed contract is often \emph{smaller than the prose it replaces} (banking: 1{,}063--1{,}986 characters vs.\ 1{,}870 for the intent-only prompt): the table substitutes for coordination guesswork.

\textbf{S4 --- Monitor generation.} Each local type compiles mechanically to its EFSM; the runtime monitor for role $r$ is an interpreter of $M_r$ with a state pointer and a value store for guard evaluation---a small generated Python program, deterministic, $O(1)$ per event, no LLM call. Verdicts: \emph{allow} / \emph{repair} / \emph{block}, plus \texttt{choice\_guard\_violation}. In \emph{gate} mode, off-contract sends are rejected \emph{pre-delivery} and the role is re-prompted with its enabled actions; by T4 these purely local gates jointly enforce $\gtG$.

\textbf{S5 --- EFSM scheduling.} The same automaton knows whose turn it is. Instead of polling every agent every round (each idle poll a full LLM call answering \textsc{wait}), the scheduler prompts only roles enabled at the current protocol state, and tells sender-only roles so at the point of decision. This targets the dominant cost (the turn loop: $\sim$4.5 calls per delivered message unscheduled) and the dominant non-safety failure (liveness stalls, \S\ref{sec:results}).

\subsection{The typed event log and judge-free metrics}\label{sec:log}
The monitor emits, per message, a typed event: role, protocol position, branch choice, gate verdict, guard outcomes, goal-marker state. Over this log, evaluation is counting, not judging: conformance rate (allowed/repaired/blocked, per message---Completion-under-Policy computed mechanically); path distribution (each run reduces to a named path through $\gtG$; a retry-loop entry rate jumping 9\%$\to$40\% after a change is a regression no golden trajectory could encode); goal progress (read off $\checkmark\varphi$ markers---the deterministic replacement for LLM-judged ``milestone completion,'' and the only metric class that sees an agent looping in place); detour cost (events $\div$ the compile-time witness's shortest accepting path); repair burden (gate interventions over time). All are bit-reproducible: re-running the evaluators on a run directory yields identical numbers although the measured agents are stochastic.

\section{Evaluation Methodology}\label{sec:method}
\subsection{Defining conformance and goal achievement}
\textbf{Conformance, three levels.} Let $T$ be a run's trace. \emph{(C-struct)} Structural: every message of $T$ is accepted by the corresponding monitor EFSM---equivalently $T$ is a path through the validated $\gtG$ (deterministic, per-message). \emph{(C-refine)} Refinement: all payload and choice-guard predicates hold. \emph{(C-sem)} Semantic: payload content is faithful to the task; only this residue may require an LLM judge, run batched, sampled, calibrated on human labels, and stored apart from the deterministic verdicts. Design principle: shrink the judged surface to what is genuinely semantic.

\textbf{Goal achievement.} A run achieves its goals \emph{strictly} iff every goal marker is satisfied, in protocol order and in the \emph{same attempt}, including the \textsc{final} marker, under three critical-property checks: \textbf{C1 data provenance} (reported values trace to real inputs), \textbf{C2 context completeness} (required reads precede the decision, by timestamp), \textbf{C3 authorization before irreversible acts} (\texttt{approve} precedes \texttt{file}, by message order). Relaxed rungs (role-pair, semantic) are reported alongside. We additionally report \textbf{clean-goal completion (CGC)}: the fraction of trials that reach the goal with \emph{zero} violations of any severity. GCR asks ``did it get there''; CGC asks ``did it get there without breaking anything on the way''---the gap between them is precisely the lucky-run phenomenon that pass-rate metrics hide. One sampled human-labeled outcome measure stays \emph{outside} the family as an anchor: a perfectly conformant system can execute a badly conceived choreography, and divergence between excellent protocol metrics and unhappy users indicts $\gtG$, routing to re-drafting with the human, not to patching agents.

\subsection{Consequence-graded violations (S0--S4)}
Raw ``violation $=$ deviation from our protocol'' is circular---baselines that never saw the protocol fail by construction. We therefore (i) align observed message labels semantically before judging, then (ii) grade each surviving deviation by \emph{consequence} against the partial order that has reasons (data and authority dependencies), annotated with irreversibility: \textbf{S0} benign reorder/dialect (not counted); \textbf{S1} waste (duplicates, polling; cost only); \textbf{S2} skipped obligation / broken order; \textbf{S3} never terminated; \textbf{S4} \emph{irreversible act before its authorization} (disaster class). The taxonomy is itself validated: across every arm of every run, $P(\text{goal failure}\mid\text{attempt had S2+})=100\%$, while benign deviations did not predict failure---the grader measures harm, not difference.

\subsection{Arms and fairness}
All arms share the intent, role descriptions, model, step and retry budgets; the canonical validated type judges every arm (agents in A never see it). \textbf{A}: intent only. \textbf{B}: A $+$ the full validated global type pasted as text. \textbf{C}: A $+$ only the role's projected local contract (observer monitor). \textbf{C-min}: C with a one-line-per-transition contract. \textbf{C+}: C plus the \emph{enforcement gate} (and, where stated, the scheduler --- the full \stjp{} stack). Experiments were pre-registered: predictions written before running, graded after (\S\ref{sec:honest}).

Five findings, in order of importance (Table~\ref{tab:grand}). \textbf{(1) B and C+ tie on outcome---and B is cheaper.} Both reach 100\% with zero harm; B does it at 24.4k tokens per success vs.\ 79.5k. They differ in \emph{kind}: B relies on the model \emph{choosing} to follow pasted text (no mechanism); C+ blocks wrong messages mechanically (it intercepted five premature \texttt{Approval} sends this run). \subsection{Scheduler ablation: winning the cost back ($9\times$)}
On a shared decentralized runtime (finance, gpt-5.4, $n{=}10$; run 2026-07-02), protocol-as-text costs 120k tokens per delivered goal at 41.8 calls/trial, because every role re-reads the whole protocol and idle roles are polled every round. Lean contract $+$ gate: 38k at 34.0 calls. Full \stjp{} (adding the EFSM scheduler): \textbf{13.3k tokens per goal, 11.4 calls, 32s/trial}---$9\times$ cheaper than B and $-73\%$ calls, at identical 100\% GCR and zero disasters across all three (Fig.~\ref{fig:results}d). Tokens-per-call stayed flat (1.12k vs.\ 1.07k), so the entire saving is \emph{fewer calls}: the mechanism, not verbosity effects. The gate rejected 10--12 off-contract send attempts per trial even in succeeding runs---enforcement measured as work done, not decoration.

\subsection{Banking: S4 in dollars; and the statically caught deadlock}
On the live-drafted 5-role banking pipeline (gpt-4o, $n{=}2$), intent-only agents \emph{moved money before authorization} twice in six attempts, with nothing in their own stack flagging it; typed arms reached 100\% GCR with zero S4 (and their eight raw monitor flags collapsed under grading to a single harmless S2). Separately, a two-role trading protocol seeded with the classic circular wait (Desk waits for Executor's ready; Executor waits for Desk's request) stalls every bare run forever; Scribble rejects the global type \emph{before any agent runs}, at zero token cost, and the repaired protocol executes at 100\%. No runtime guardrail, judge, or benchmark can offer this: they all require the failure to occur at least once. Testing observes runs; typing quantifies over all of them. The same circular-wait pattern is not merely authored: adapting the buyer and seller agents of a real open-source negotiation benchmark \citep{agenticpay} (MIT) and adding an escrow settlement layer reproduces the deadlock, and a bounded live run across three model tiers shows it is capability-invariant while the validated escrow-first protocol completes (full metered benchmark pending).

\subsection{Pre-registered predictions, graded --- including the ones we got wrong}\label{sec:honest}
\textbf{P1 (falsified twice, differently).} We predicted unenforced global text (B) would slip below 100\%: under gpt-4o it did (40\%); under gpt-5.4 it did not (100\%, \S\ref{sec:grand}). What survives both outcomes: the gate rejected real off-contract attempts in \emph{every} gated run---the model tries to stray; the boundary catches it---and enforcement value is a decreasing function of model strength while the danger of \emph{no protocol at all} is an \emph{increasing} one (22 vs.\ 4 disasters). \textbf{P2 (confirmed beyond margin).} Scheduling saves $\geq$60\% tokens vs.\ lean$+$gate: measured $-65\%$ tokens, $-66\%$ calls. \textbf{P3 (half-confirmed).} Projection alone does not beat global text; projection \emph{with enforcement and scheduling} dominates it on the same runtime. \textbf{P4 (deferred).} Orchestrator-cost comparison was inconclusive (infrastructure issues on the group-chat arm). \textbf{P5 (confirmed).} Scheduling cuts calls without inflating tokens per call. We report these grades because a verification paper that hides its own falsified predictions has misunderstood its subject.

\subsection{Two systems lessons}\label{sec:lessons}
\textbf{Liveness cannot be enforced by observation.} The dominant failure of one gate run was inaction: a role whose whole contract is one \textsc{send} answered \textsc{wait} repeatedly; a monitor judges events, and a \textsc{wait} emits none. A contract can make every action that happens correct; it cannot, by observation alone, make an action happen. The fix is structural---the projection already contains the schedule (S5). \textbf{Tone is part of the contract surface.} An imperative liveness nudge (``WAIT is OFF-CONTRACT here; act now,'' with a mojibaked em-dash) made gpt-5.4 \emph{refuse outright} in both nudged roles, stalling the arm at zero events; the same content rephrased descriptively (``Protocol status: \ldots the available action at this state is \textsc{send} \texttt{ExpenseData}\ldots'') was followed 5/5 first-attempt. Runtime governance text must be neutral, ASCII-safe, and informative; stronger models police the difference.

\section{The Validation Suite at $n{=}100$}\label{sec:bench}
The pilot evidence of \S\ref{sec:results} was concept-scale ($n{\leq}10$, one model family, friendly agents, an untested checker). We therefore designed a component-isolating suite---one experiment per component, each pre-registered before running---and executed it at $n{=}100$; the E3 capability sweep is now measured across three Claude tiers on both tasks. Everything below is \textbf{measured} (repository, with a per-number data map), fully terminal (0 abandoned trials after the forensic audit), and its total compute cost was under \$100.

\textbf{E0 --- Testing the testers first.} A 40-trace verdict corpus with hand-derived expected verdicts (clean traces, every severity class, concurrent interleavings on independent channels, unevaluable-guard silence, the \texttt{Approval(False)} edge case). Monitor and grader score 40/40---and, instructively, three of the \emph{hand derivations} were wrong and corrected against the semantics; the instruments were right. All downstream experiments inherit this validated referee. An integration stress harness additionally exercises all five pipeline stages 100 times ($2105/2110 = 99.76\%$ checks passing; the five failures are a FIFO-swap mutation that yields another well-formed protocol on acyclic inputs, which Scribble \emph{correctly} accepts).

\textbf{E1 --- Mutation testing the checker (Fig.~\ref{fig:projected}a).} Seven mutation operators over a 100-protocol corpus (2--10 roles, four topologies). The pre-registered prediction separated two families, and the data respected the split: \emph{well-formedness} defects are detected at 95.1\% (226 mutants; undeclared role and flipped-choice-subject at 100\%, branch asymmetry---a role acting on branch knowledge it cannot have---at 82.5\% raw) with 0/100 false positives; adjudicating the eleven branch-asymmetry survivors found seven to be \emph{accidentally valid} mutants (correct acceptances) and four to share one reproducible genuine gap---the deletion empties a branch, projection silently drops it, and an idle role is left unconditionally awaiting a message the chooser may never send, a knowledge-of-choice deadlock the checker misses---re-scoping detection to real defects at 92.9\% and converting a bare percentage into a named limitation; \emph{reordering} operators (400 mutants) are flagged at only 0.8\%---correctly, because reordering an acyclic protocol usually produces a different but valid protocol. Detecting \emph{intent} drift under valid reorderings is the E5 scorer's job, not the type checker's; conflating the two is how benchmarks overstate their tools. The evaluation standard here is deliberately the two-sided \emph{preciseness} discipline of \citet{chen17,chen24}---soundness (nothing bad accepted) and completeness (nothing good rejected) measured together---transplanted from characterizing a subtyping relation to auditing a checker implementation.

\textbf{E2 --- Adversarial exfiltration (Fig.~\ref{fig:projected}b).} Twelve prompt-injection templates $\times$ 100 trials against a finance protocol with a confidential payload and an ExternalAuditor role that legally receives only the public summary. No guard: 0\% blocked. An honestly tuned keyword-and-role rule list: 41.7\%, evaded by encoding, synonyms, typos, split payloads, and innocuous relabeling. The structural gate: 91.7\%---it blocks every attempt to use an illegal (label, recipient) edge regardless of disguise, because legality is checked against the type, not the wording; the surviving 8.3\% is exactly one template that smuggles the payload \emph{inside} a legitimately-labelled message on a legal route, which structure cannot see. Gate plus value-refinement guards: 100\%. Each mechanism catches precisely the class the theory says it catches.

At $n{=}100$ under concurrent polling, arm B reaches the goal in 100\% of \texttt{revenue\_audit} trials while committing the forbidden act---filing before approval---in \emph{95 of 100}. \textbf{E5 --- Translation fidelity (Table~\ref{tab:transfid}).} The deterministic half is done: the EFSM-bisimulation scorer that will judge LLM-drafted protocols against expert gold (a known-correct reference protocol) classifies 300/300 pairs (identical / semantics-preserving reformat / seeded mutant) correctly. The LLM-dependent half---first-draft validity, repair rounds, guard co-emission---remains the declared open piece of the intent-to-protocol translation step (called ``the seam'' below) and is measurement-pending; the scorer's validation is what makes those forthcoming numbers trustworthy.

\textbf{E6 / E7 --- Scaling and portability.} The roles sweep (Fig.~\ref{fig:projected}d) confirms the pre-registered shape on a structural proxy: global-text coordination cost grows super-linearly (every role re-reads a growing protocol under poll-all), \stjp{} near-linearly (each role sees only its slice; one enabled poll per step), with the ratio widening from $9.2\times$ to $17.1\times$ between 2 and 10 roles; token-metered curves are pending. For portability, the standalone generated monitor agrees with the in-process monitor on 59/59 testable canonical traces (41 choice-only protocols yield no meaningful canonical trace; the claim is scoped to testable ones), and a LangGraph adapter executing the ladder protocols as StateGraphs agrees with the native monitor on 14/14 clean-and-injected checks, consistent with T4's placement of enforcement at message boundaries; the skill-runtime third harness is pending.

\textbf{Pre-registration, graded; and the audit that changed a number.} All eight v2 predictions were registered before running and graded after: E0, E1 (with the reordering-acceptance clause), E2 (with the predicted legal-route smuggle), E4, E5-scorer, E6-proxy, and E7-agreement \textsc{confirmed}; E3, registered pending, is now measured with its capability clause confirmed and vendor diversity still open. A post-hoc forensic audit of the full dataset then re-verified every suspicious pattern: B's low-diversity replies were adjudicated genuine on three independent behavioral signals (three distinct trace shapes, including a five-trial serialized branch no script would produce); intent-only's zero disasters trace to accidental serialization, its 591 violations all named duplicate-send waste; and the audit surfaced one material bug the checklist had not targeted---22 of 1{,}200 trials were abandoned mid-play rather than failed, 18 of them depressing one gated arm's reported liveness to 79\% against a true $\sim$97\%. All 22 were driven to termination and every table regenerated; the 19 \emph{genuine} gated-arm deadlocks were left standing, because replaying real failures until they pass is p-hacking. We also log the integrity incidents scale surfaced: subagent role-players caught writing auto-responder scripts instead of reasoning per poll (detected by a \texttt{malformed==calls} signature and self-confession, discarded and replayed), a round-numbering bug that could mask disasters (fixed with causal detection), and a false-positive disaster check on enforcing arms (fixed mid-analysis); the final audit verified all 1{,}200 ladder trials against raw state files with zero remaining fraud and zero non-terminal trials. We report these because a verification paper that does not verify its own experiments has misunderstood its subject.

\section{Three Typed Extensions, Measured}\label{sec:ext}
The validation suite certifies what \stjp{} ships; this section extends what it can \emph{catch}, \emph{build-check}, and \emph{survive}, with one prototype per session-type result: guards \citep{bocchi10} $\to$ stateful properties \citep{chen12} $\to$ lawful slack \citep{chen17,chen24} $\to$ typed recovery \citep{viering18}. Each prototype passed a hand-derived verdict corpus (12/14/12 traces) before its benchmark ran; all predictions were pre-registered and graded, including the one we grade against ourselves. These are failure-mode coverage additions, not speedups: none moves the core cost or safety numbers of \S\ref{sec:results}--\S\ref{sec:bench}, and each write-up names the shipped system's blindness as the motivation, not a planted defect.

\textbf{E8 --- Stateful session invariants (the ledger).} Per-message guards see one message; a budget of \$10k with a per-debit limit of \$5k is overrun by three \$4k debits that every shipped check individually approves---a violation class the gate is blind to \emph{by construction}. Following \citet{chen12}, the \texttt{.refn} sidecar gains \texttt{state}/\texttt{on}/\texttt{invariant} clauses; the generated monitor carries a session store, applies updates on accept, and checks invariants before delivery (centrally at the gate in v1, deferring the per-endpoint distribution---where the asynchronous machinery of \citet{chen12} becomes essential---to future work). Measured: on 100 seeded traces, the shipped arm detects 0/50 overruns, the ledger flags 50/50 \emph{at the exact crossing message} with 0/50 false positives; in live trials (a \$12k procurement against the \$10k budget, $n{=}15$/arm) the shipped gate paid 16 over-budget debits undetected, the ledger-observe arm flagged 15/15, and the ledger \emph{gate} paid exactly \$10{,}000 in every trial while still reaching the goal---the Treasurer, re-prompted with the invariant text, downsized or refused. Cost: a few sidecar clauses. Bounds: invariants are checked dynamically (not SMT-proved) and centrally.

\textbf{E9 --- Lawful slack (precise subtyping), a win and a boundary.} Two deliverables from one theorem. \emph{(2a)} A coinductive subtype checker (output covariance, input contravariance, send-polarity) makes $\text{lean} \leq \text{projection}$ a decidable \emph{build step}: contract compression is now verified rather than asserted, and the checker catches a seeded lean contract that drops a required send. \emph{(2b)} A tolerant gate implementing the one decidable, provably safe relaxation---independent-receive output anticipation---behind a default-off flag. The mandatory control holds: 0/50 illegal sends admitted, so tolerance never widens the gate beyond the precise relation. The deadlock replay, pre-registered as uncertain (``$\geq$5/19 rescued''), returned the more interesting answer: 16/19 deadlocks do contain a rejected send that subtyping proves type-safe (the strict gate \emph{is} stricter than safety requires), yet all 16 share one root cause---the Buyer never sent \texttt{Deposit}; the rejected sends were other roles correctly waiting on an absent participant, so admitting them rescues nothing---and live reruns needed 0 anticipations ($n{=}15$/arm, both gates 15/15). The boundary result is worth stating plainly: on authorization-laden protocols, message orderings \emph{encode obligations} (pay-before-ship), so the slack that subtyping licenses at the type level is precisely what one does not want to spend; lawful slack is real, and this domain is where you leave it unspent. The type theory and the deployment answer diverge, and reporting both is the point.

\textbf{E10 --- Typed crash handling (no session left in limbo).} The audit's 22 non-terminal trials were untyped crash failures; following \citet{viering18}, a \texttt{.fail} sidecar declares per-region handlers, the validator statically checks \emph{coverage} (every state $\times$ crashable role handled or explicitly escalated), \emph{handler safety} (handler bodies are re-validated as mini global types through the same Scribble pipeline), and \emph{recoverability} (every crash reaches a typed terminal); the scheduler---already the coordinator---gains a poll-timeout crash detector, and monitors swap to pre-generated handler EFSMs. Measured: the full crash-point grid ($14$ cells) leaves the shipped system in limbo 14/14 and \stjp{}+CF at 14/14 typed terminals (9 degraded, e.g.\ \texttt{Refunded}, 5 clean aborts) with 0 disasters; all 19 real recorded deadlocks replay to typed terminals; live flaky-role trials ($n{=}15$, one role goes silent per trial) resolve 15/15. Critically, the new checker is held to the E1 preciseness standard: 5/5 seeded unsafe handlers are rejected statically---including a settlement-shortcut handler whose ``recovery'' would bypass authorization, exactly the class a naive recovery mechanism would smuggle in. ``No session left in limbo'' is now a compile-time property, and the audit finding that embarrassed us most is the one this section closes.

\section{Limitations}\label{sec:limits}
\textbf{The intent-to-protocol translation step.} Intent$\to$Scribble and contract$\to$skill rendering pass through an LLM; the validator rejects structurally invalid drafts, but a valid-yet-unintended protocol survives (validated $\neq$ intended). Shipped mitigations: human endorsement of $\gtG$; the outcome anchor. Planned: benchmark item 5 and a fine-tuned translator. \textbf{Scope of the theorems.} T3 is per-session; interleaved sessions need compositional/hybrid extensions; dynamic role join/leave needs \citep{denielou11}; our experiments fix the role set. \textbf{Expressiveness.} Open-ended brainstorming resists protocolization, and forcing it into a type would be dishonest; \stjp{} targets the coordination-critical, authorization-laden regime. Probabilistic session types are a candidate middle path. \textbf{Statistics and scope of the $n{=}100$ suite.} The live gpt-4o/gpt-5.4 finance runs remain $n\leq10$ per cell; the $n{=}100$ suite fixes the statistical power but substitutes cheap subagent role-players (ladders) or contract-following strategies (E4) for frontier models, and plays each trial's roles from one subagent's per-role views rather than truly independent agents. The E3 sweep now spans three Claude tiers on both tasks but lacks a non-Claude frontier point (an access limitation), so vendor generality of the capability curve remains open; the tone finding (\S\ref{sec:lessons}) especially needs cross-model replication. \textbf{Grader and guards.} The post-hoc severity grader is payload-blind at milestones (\texttt{Approval(False)} currently counts as the authorization milestone; the monitor-level guard closes this, the grader upgrade is pending); choice-guard authoring is manual (the drafter should co-emit the sidecar). \textbf{Extension bounds.} The crash-handling roadmap item is now implemented and measured (\S\ref{sec:ext}), with its own bounds: a \texttt{.fail} sidecar must be authored per protocol, and the live crash model is a role going silent. The session ledger checks invariants dynamically and centrally, not by SMT proof or per-endpoint distribution \citep{chen12}; the tolerant gate is provably safe but, on obligation-encoding protocols, deliberately left off by default (\S\ref{sec:ext}). \textbf{Monitor correctness} is validated (theory-matching, regression tests, manual audits), not mechanized.

\section{Conclusion}\label{sec:conc}
When skills became the programming language of multi-agent systems, the field skipped the compiler and went straight to observability. \stjp{} retrofits the missing stage: intents compile into a protocol algebra with a forty-year pedigree; faulty choreographies are rejected before a single token is spent; and the artifacts agents actually receive---lean local skills, generated monitors, guards, a protocol-driven scheduler---are projections of a proven object rather than prose approximations of a hopeful one. A 1{,}200-trial validation suite replicates the shape at $n{=}100$: observing arms accumulate real violations that goal-rate metrics hide, enforcing arms show zero by construction, and the checker itself survives mutation testing with zero false positives. The empirical shape is what compilation always buys: the same guarantees, then made cheap ($9\times$ on cost-to-goal once the projection is allowed to \emph{drive} the runtime), plus a class of properties---deadlock-freedom, goal reachability, authorization ordering, guarded branching, recipient confidentiality---that no amount of runtime watching can provide, because they quantify over runs that never happen. Prompts move probabilities; only checks move guarantees. The open seam---natural language in, formal protocol out---is real, measurable, and, we argue, the right next benchmark for the field.

\section{A validated protocol and its guard sidecar}\label{app:scr}
Global type (Scribble) for the finance case, abbreviated:
{\small\begin{verbatim}
global protocol Finance(role Ingestor, role RevenueAnalyst, role ExpenseAnalyst,
                        role TaxVerifier, role Auditor, role Writer) {
  RawRevenueData(Payload)  from Ingestor to RevenueAnalyst;
  ExpenseData(Payload)     from ExpenseAnalyst to RevenueAnalyst;
  choice at RevenueAnalyst {
    HighRevenueNotification(Analysis) from RevenueAnalyst to Auditor;
    AuditReport(Findings)             from Auditor        to TaxVerifier;
  } or {
    StandardRevenueNotification(Analysis) from RevenueAnalyst to TaxVerifier;
  }
  Approval(Decision)       from TaxVerifier to Writer;   // goal marker
  FinalReport(Document)    from Writer to Ingestor;      // goal marker, FINAL
}
\end{verbatim}}
Choice-guard sidecar (\texttt{.refn}; asserted-MPST style; stock Scribble untouched):
{\small\begin{verbatim}
[choice at RevenueAnalyst]
when:    float(RawRevenueData) > 50000
require: HighRevenueNotification
over:    StandardRevenueNotification

[payload Approval]
assert:  Decision == 'true'  =>  next in {FinalReport}   # denied-but-debited hole
\end{verbatim}}
The guard travels through four stages: \emph{define} (sidecar) $\to$ \emph{state} (compiled into the contract at the decision state) $\to$ \emph{check} (value-tracking monitor; verdict \texttt{choice\_guard\_violation}) $\to$ \emph{enforce} (gate). Offline replay verification fires on high-revenue$+$standard-branch and on the symmetric case, stays silent on correct branches and on unevaluable guards.
