MULTIPARTY SESSION TYPES × LLM AGENTS

From one user intent to a deadlock-free multi-agent contract — and the receipts to prove agents follow it.

STJP turns a natural-language intent into a Scribble global type, validates it (no deadlocks, no infinite loops), projects it into per-role local contracts, and drops a static Python monitor — not an agent — next to every agent. Below: the same task, the same model, three levels of protocol information. Every trace shown is a real run.

05 · How we score "the ultimate goal"

  
Benchmark design — four gated layers

  
Each layer gates the next: a run that never terminates cannot score on goals;
  goals scored out of order earn no ordered credit. Headline = three dials — did they deliver
  (GCR), did they follow the contract (path adherence), at what price (cost of success).
  Full spec: docs/BENCHMARK_DESIGN.md.

  

    
LAYER 0 · GATE
Liveness

      
The MPST guarantee, measured directly.

      
  • Termination rate — terminal label within budget
    • 
      
  • Stall profile — deadlock vs livelock vs budget-exhausted
    • 
      
  • First-stall depth

    
LAYER 1 · OUTCOME
Goal achievement

      
Intermediate goals + one final goal, anchored to protocol messages with payload predicates.

      
  • GCR — all applicable goals + final goal, same attempt

  • Strict / role-pair / semantic rungs

Path adherence

      
"Some paths must be followed."

  • First-violation depth
