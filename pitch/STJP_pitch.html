STJP · an agent-interaction compiler

A compiler for the interactions between AI agents.

STJP (session-typed judge panel) ensures the interactions among your agents are correct by construction: it compiles your intent into a protocol that’s mathematically proven deadlock-free before any agent runs, gives each agent its own contract, and enforces it live with deterministic code — twenty years of session-type theory, finally pointed at agents.

01
01 / The blind spot

Everyone writes the spec. Nobody can prove its correctness.

The whole industry is racing to govern AI agents with skills, specs, and markdown files that say what each agent may do. But almost nobody asks the question underneath: how do you know the interactions you wrote down are correct? A human cannot tell by reading. Each sheet can look perfect alone, while the way they combine hides a fault no eye can catch — a freeze, a missed message, a step in the wrong order. Today’s validators check a file is well-formed; none check that the conversation it describes can actually happen.

An orchestrator runs a “ship the fix” job with three subagents. The Coder won’t push its patch until the Reviewer signs off. The Reviewer won’t sign off until tests are green — so it waits for the Tester. The Tester can’t run until the patch is on the test branch — so it waits for the Coder. Each sheet reads fine alone; together they form a circle where everyone waits to receive before anyone will send. The run just hangs — no crash, no log.
This isn’t hypothetical — it is a documented multi-agent failure mode (MAST, Cemri et al., UC Berkeley, 2025).
the failure STJP rules out
A human can’t read three sheets and see a circular wait. Logical proving systems can.

02 / The fix is proven

Multiparty Interactions Governance: MPST → Scribble → STJP

Computer science settled “is this multi-party conversation safe?” two decades ago, with Multiparty Session Types (MPST). Given the rules of a conversation among several parties, MPST proves — as a theorem, before anything runs — that the protocol is deadlock-free, communication-safe, and protocol-faithful. With session subtyping it proves one thing more: you may swap one participant for another — a cheaper model, a different vendor’s agent — as long as its behaviour refines the contract, and every guarantee still holds.

✓ deadlock-free ✓ communication-safe ✓ protocol-faithful ✓ safe agent swaps

MPST then projects the proven-correct global protocol onto each participant as a local type — a precise contract for that one role. Traditionally those guided hand-written code. STJP’s move is to project them onto each agent instead: every agent gets a behaviour guide carved from a protocol already proven safe, with the safety locked in at compile time — not hoped for at runtime.

Scribble is the MPST type checker: it validates a global protocol against MPST’s typing rules, accepting it only if it is provably sound. STJP wires this into a loop anyone can drive:

  1. You give STJP the intent of a task, in plain language.
  2. An LLM translator turns that intent into a candidate global type.
  3. Scribble validates it. If it fails — a deadlock, an impossible step — the translator re-drafts, and Scribble checks again, until it passes.
  4. The validated global type is projected into a local contract per agent. Only then does any agent run.
03
03 / Enforced live

Enforced at runtime — with code, not another AI

The compile-time validator is the Checker (Scribble). Three more roles run at runtime — and the one that says no is deterministic code, not a model that could flail.

runtime · gate
Monitor

Holds each agent’s local contract and checks every message both ways — may this sender send this type with constraints met? would this receiver break its contract by accepting? On any violation it stops the message.

deterministic code — not an agent
runtime · oversight
Critic

Reads across all the Monitor’s traces for the broader view: do they, as a whole, stay inside the global type? Is any security or policy line crossed that a single message wouldn’t reveal?

runtime · repair
Revisor

When the Critic flags something, it revises the protocol or contract. The rails get corrected by design — not by hoping the models improve.

04 / Where it fits

Where STJP fits in the governance stack

Agent governance stacks three layers over a single agent — format, semantics, behavior. Each is checked alone, and nothing proves the three agree.

LAYER 1
Format conformance

Does its spec file parse? Solved — skill validators, schema checks.

LAYER 2
Semantic soundness

Does the spec match the tool’s real capabilities and boundaries? Not solved.

LAYER 3
Behavioral verification

Does spec + runtime + agent behave as claimed? Emerging — BeSafe-Bench, ST-WebAgentBench.

THE MISSING PROOF — STJP
Making all three layers agree

A contract proven correct by theory (Layer 2) and enforced by deterministic runtime code (Layer 3), plus the vertical proof that connects all three layers into one guarantee.

05 / Why now

Interaction-level coherence, assured by design

Can’t we just trust the agents to sort themselves out? No — and the evidence says so twice.

Empirically

Most failures are coordination, not capability.

Why Do Multi-Agent LLM Systems Fail? annotated 1,600+ traces across seven frameworks: the bulk of failures come from specification and coordination between agents — and fixing them needs structural redesign, not better prompts.

MAST · Cemri et al. · arXiv:2503.13657
Theoretically

Incoherence is inherent, and scale won’t save you.

The Hot Mess of AI shows that the longer agents reason and the more steps they take, the more their failures become inconsistent flailing — and bigger models don’t reliably fix it.

ICLR 2026 · arXiv:2601.23045
Adaptive learning is a wish. Structural design is a guarantee.

So “the agents will get better on their own” is not a lever you can pull. The dependable one is governance by structural design, applied where agents actually meet: their interactions. STJP delivers exactly that — coherence assured at the interaction level by design: proven at compile time, enforced live.

03
03 / Why it’s different

What ASSERT and ACS don’t do

Microsoft just shipped two strong pieces for agent safety. Both are useful. Neither looks at the thing STJP looks at — the conversation between agents.

tests · after the fact
ASSERT

A tester. You give it the rules you care about; it invents tricky situations, runs your agents through them, and scores how they did. Good for finding problems by trial — but it only catches what its tests happen to hit, and only after a run. It can’t promise a freeze won’t happen.

guards · one agent
ACS

A guard on a single agent — watching what goes in, what the model says, what tools it calls, what comes out, and allowing, blocking, or editing each step. Excellent for “should this one action be allowed?” But it only ever sees one agent, one moment at a time. It has no picture of the whole conversation, so it can’t tell whether the group will freeze.

proves · the whole conversation
STJP

Checks the conversation itself — all the agents together, before anything runs — and proves the back-and-forth can finish. It answers the question underneath the other two: can this group actually get through the conversation? Then it enforces that answer, live.

ASSERT crash-tests the car. ACS is the seatbelt in each seat. STJP makes sure the roads connect so the cars don’t all gridlock — the problem neither of the others was looking at.

They don’t compete — you’d want all three: ASSERT to test, ACS to guard each agent, STJP to make the conversation sound. STJP can even hand its per-agent checklists straight to ACS to enforce. The freeze is exactly the failure ASSERT can miss and ACS can’t see — and it’s the most expensive one, because nothing breaks; the job just hangs. STJP is the only piece that prevents it before it can start.

06 / The ecosystem

Where STJP sits next to Aleph and Arcade

Arcade

Keeps an agent’s reach into the outside world safe — Gmail, Stripe, GitHub. Not how agents talk to each other.

Aleph

Gets one hard answer right and proves it. Not the back-and-forth among several agents.

STJP

Proves the conversation that ties the agents together is sound — and keeps it that way, live.

The takeaway
STJP proves the conversation between your agents — and keeps it sound, live.
Arcade guards each agent’s connections to the outside world; Aleph proves a single hard calculation. STJP is the missing layer — the one that proves the interactions between agents can’t break.
